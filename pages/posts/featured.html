The complete guide to CSS Grid — from first principles to production

CSS Grid is the most powerful layout system available in CSS. This guide walks through every property, every pattern, and every gotcha you'll encounter building real layouts — from a single-page portfolio to a full magazine-style template.

MA
Maria Alcantara
Senior CSS Developer
Published May 12, 2024
Read time 12 min
Level Intermediate

Before CSS Grid existed, front-end developers built layouts using floats, clearfix hacks, and prayers. Then Flexbox arrived and things got better — but Flexbox is a one-dimensional tool. CSS Grid is two-dimensional. It controls rows and columns at the same time, which changes what's possible at a fundamental level.

💡
The key mental shift Flexbox decides how items arrange themselves. Grid lets you decide where every item goes. Start with Grid for the page-level layout, use Flexbox inside components.

How CSS Grid actually works

When you write display: grid on a container, you've created a grid formatting context. The container becomes the grid, and its direct children become grid items. Nothing changes visually yet — you haven't defined any columns or rows.

The next step is defining your tracks. A track is a single column or row in the grid. You define them with grid-template-columns and grid-template-rows.

.container {
                display: grid;
                grid-template-columns: 200px 1fr 1fr;
                grid-template-rows: auto;
                gap: 1.5rem;
              }

This creates a 3-column grid: a fixed 200px sidebar, and two columns that share the remaining space equally. The 1fr unit — "one fraction of available space" — is the most important unit in Grid. It handles all the math you used to do manually.

The fr unit — the most important Grid concept

Before fr, making two columns share space equally meant writing width: calc(50% - 12px) and crossing your fingers. With fr, you write 1fr 1fr and Grid handles it — gap included.

The repeat() function

Writing 1fr 1fr 1fr 1fr for a 4-column grid works but gets tedious fast. repeat() is shorthand:

/* These are identical */
grid-template-columns: 1fr 1fr 1fr 1fr;
grid-template-columns: repeat(4, 1fr);

/* Auto-responsive: as many columns as fit, minimum 200px each */
grid-template-columns: repeat(auto-fit, minmax(200px, 1fr));

That last line — repeat(auto-fit, minmax(200px, 1fr)) — is one of the most powerful one-liners in all of CSS. It creates a responsive grid that adjusts its column count based on available space, with no media queries required.

"The best layout systems make the right thing easy and the wrong thing hard. CSS Grid makes responsive layouts easy and manual column math unnecessary." — Rachel Andrew, CSS Grid contributor

Placing items precisely

Grid's real power comes from explicit placement. You can tell any item exactly where to sit in the grid using line numbers, named areas, or span keywords.

/* Place an item from column line 1 to 3, row line 1 to 2 */
.hero {
  grid-column: 1 / 3;
  grid-row: 1 / 2;
}

/* Span shorthand — span 2 columns from current position */
.wide-card {
  grid-column: span 2;
}

Named grid areas

For complex page layouts, named areas are more readable than line numbers. You draw the layout in ASCII art, then assign items to named zones:

.page {
  display: grid;
  grid-template-areas:
    "header header header"
    "sidebar main    main"
    "footer  footer  footer";
  grid-template-columns: 240px 1fr 1fr;
  grid-template-rows: auto 1fr auto;
  min-height: 100vh;
  gap: 0;
}

.site-header { grid-area: header; }
.sidebar     { grid-area: sidebar; }
.main        { grid-area: main; }
.site-footer { grid-area: footer; }

3 patterns you'll use on every project

Responsive card grid

The classic use case — a grid of cards that collapses from 4 columns to 2 to 1 as the screen narrows, without a single media query:

.card-grid {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(280px, 1fr));
  gap: 1.5rem;
}

Holy grail layout

The infamous "header, sidebar, main, footer" pattern that once required nested floats and JavaScript is now 12 lines of CSS:

.layout {
  display: grid;
  grid-template:
    "header" auto
    "main"   1fr
    "footer" auto
    / 1fr;
  min-height: 100vh;
}

@media (min-width: 768px) {
  .layout {
    grid-template:
      "header  header" auto
      "sidebar main"   1fr
      "footer  footer" auto
      / 240px 1fr;
  }
}

Magazine layout

Items of different sizes, positioned precisely — the kind of layout that was nearly impossible before Grid:

.magazine {
  display: grid;
  grid-template-columns: repeat(6, 1fr);
  gap: 1rem;
}

.feature { grid-column: span 4; grid-row: span 2; }
.aside   { grid-column: span 2; }
.strip   { grid-column: 1 / -1; }  /* Full width — -1 = last line */
  • Use 1 / -1 as a shorthand for "span the full width" — -1 always refers to the last grid line.
  • Combine grid-column: span N with auto-fill to let items reflow naturally.
  • grid-auto-rows: minmax(100px, auto) gives implicit rows a minimum height, preventing collapsed cells.

3 mistakes that trip everyone up

Mistake 1: Applying Grid to the wrong element. Grid only controls direct children. A grandchild element is not a grid item and won't respond to grid-column or grid-row — you need a nested grid for that.

Mistake 2: Confusing auto-fit and auto-fill. Both create as many columns as possible. The difference is what happens with empty space: auto-fill keeps empty tracks, auto-fit collapses them. For card grids, you almost always want auto-fit.

Mistake 3: Forgetting about implicit rows. When you place an item outside your defined rows, Grid creates an implicit row automatically. These default to height: auto — which is usually fine, but can cause uneven row heights. Control them with grid-auto-rows.

Browser support in 2024

CSS Grid has 97%+ global support. Every major browser — Chrome, Firefox, Safari, Edge — has supported it since 2017. There are no polyfills needed and no fallbacks required for any modern project. Ship it with confidence.
